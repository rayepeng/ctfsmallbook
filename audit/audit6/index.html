    审计小知识¶

    php自身在解析请求的时候，如果参数名字中包含” “、”.”、”[“这几个字符，会将他们转换成下划线。¶

    伪协议php://filter¶

    php://input和php://output分别代表标准输入和标准输出，php://filter其实就是一个过滤器。

    我们这么来理解:

    1
2
3
4
5
    php > $fp = fopen('php://output', 'w');
php > var_dump($fp);
resource(2) of type (stream)
php > fwrite($fp, "This is a test.\n");
This is a test.

    很明显我们向标准输出中写入了一句话，也就是说我们的此时的目的是向标准输出写一句话。

    如果要转成大写呢？

    1
2
3
    php > stream_filter_append($fp, 'string.toupper');
php > fwrite($fp, "This is a test.\n");
THIS IS A TEST.

    可以看到，只需要附加一个string.toupper过滤器就行，于是：

    1
2
3
    php > $fp=fopen('php://filter/string.toupper/resource=php://output', 'w');
php > fwrite($fp, "This is a test.\n");
THIS IS A TEST.

    php://filter/string.toupper/resource=php://output看上去很负责

    1. 我们要写入的目标是php://output
    2. 要加一个过滤器string.toupper
    3. 合在一起就是php://filter协议

    第二行的<?php exit;?>，这句话在开头增加了exit过程，导致即使我们成功写入一句话，也执行不了（这个过程在实战中十分常见，通常出现在缓存、配置文件等等地方，不允许用户直接访问的文件，都会被加上if(!defined(xxx))exit;之类的限制）。

    file_put_contents允许我们向一个文件写入一些东西，是不是可以加入一些过滤的东东？比如:

    base64

    所以我们就是向tmp.php写入数据，但是这个数据是base64之后的，我们只需解码base64即可，而这时<?php exit;?>就会面目全非了

    $filename = php://filter/convert.base64-decode/resource=tmp.php

    因为<?php exit;?>中"<、?、;、>"等符号解码时都会被忽略，所以命令就变成了→ “phpexit”。也就达到了我们绕过了这个函数。之后我们要想执行我们所想的代码，可以在phpexit这七个字符后任意加一个字母（因为base64是4个字符一组进行解码的
